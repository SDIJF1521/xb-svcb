## v0.0.23 · 完整外部 API + 酷我音乐

> 本次更新把 XB-SVCB 从单一桌面工具扩展为可由外部程序调用的本地音频工作站：FastAPI 覆盖模型、推理、作品与 Audio Editor Lite 主要流程，在线资源获取新增酷我音乐。

> [!IMPORTANT]
> v0.0.23 继续使用分卷安装包。请同时下载 `XB-SVCB-Setup.exe` 和全部 `XB-SVCB-Setup-*.bin`，放在同一目录后再运行安装程序。

### 🌐 FastAPI 外部接入

- 🔌 **由用户手动启停**：新增“资料库 -> API 接入”页面；服务默认关闭，不随软件或系统自动启动，退出 `XB-SVCB.exe` 后自动停止监听。
- 🖥️ **本机与局域网两种范围**：默认仅监听 `127.0.0.1`，需要其他设备接入时可手动切换到 `0.0.0.0`；端口支持 `1024-65535`。
- 🔁 **桌面与 HTTP 共用业务核心**：外部任务复用桌面端的模型服务、作品仓储、编辑工程和串行推理队列；API 创建的任务会出现在“我的作品”，桌面任务也能通过 API 查询。
- 🪟 **保持纯 GUI 体验**：FastAPI 与 Uvicorn 在当前 GUI 进程的后台线程运行，不额外打开 `cmd.exe`、PowerShell 或可见控制台窗口。
- 🧭 **运行状态清晰可见**：接入页显示监听地址、端口、API Key、服务状态和连通延迟，并提供真实鉴权测试。
- 🧹 **生命周期完整收口**：停止服务或退出软件会释放端口；服务运行期间禁止修改端口、监听范围和密钥，避免界面配置与实际进程不一致。
- 🌍 **浏览器工具可接入**：启用 CORS，便于本机或局域网内的 Web 工具调用；业务接口仍统一经过 API Key 鉴权。

### 🔐 接口、鉴权与数据边界

- 🔑 **统一 API Key**：除 `/health`、Swagger、ReDoc 和 OpenAPI JSON 外，所有 `/api/v1` 请求必须携带软件页面显示的 `X-API-Key`；密钥可在服务停止后重新生成。
- 🗺️ **完整接口面**：OpenAPI 当前包含 `34` 条路径、`42` 个操作，覆盖系统环境、模型、上传、任务、队列、历史、预设、作品与编辑工程。
- 🎤 **完整推理流程**：支持单模型、多模型混唱和批量创建；批量请求最多 `50` 项，并在创建任何任务前完成整批原子校验。
- 🧬 **多框架参数开放**：So-VITS-SVC、RVC、SeedVC 与 DDSP-SVC 的变调、F0、设备、扩散、speaker、index、保护、采样步数和共振峰等参数均有类型、范围与默认值说明。
- 🎙️ **SeedVC 参考音频**：单模型任务、多模型独立参数和编辑器局部重推理都可通过 `reference_upload_id` 使用已上传的目标音色参考。
- 🎚️ **Audio Editor Lite 自动化**：开放工程、音轨、素材导入、撤销/重做、波形、静音/歌词切分、人声分离、局部重推理、片段合并，以及工程/音轨/片段渲染下载。
- 📚 **作品与历史可管理**：外部程序可读取推理历史和队列、保存参数预设、查询作品、重试失败任务并下载生成的人声、伴奏或最终成品。
- 🛡️ **本机路径不外泄**：模型、任务和编辑器使用专门的 HTTP DTO；权重、配置、源音频、日志、片段文件和插件绝对路径不会直接进入响应。
- 🔗 **受控音频地址**：编辑工程中的本地音频路径转换为鉴权下载 URL；保存工程时只恢复服务端允许的字段，客户端不能借 DTO 覆盖内部路径。

### 📤 大文件上传与下载

- ♾️ **不设置应用层大小上限**：上传使用固定小块流式写盘，不把完整歌曲一次载入内存；可用容量只受数据目录磁盘空间和调用方超时限制。
- 🎞️ **常见音视频容器可用**：支持 WAV、FLAC、MP3、M4A、AAC、OGG、Opus、WMA 及常见视频容器作为推理来源。
- 🗂️ **上传随数据目录迁移**：文件保存在 `.xb_svcb/api/uploads`；迁移数据目录时会一并处理，不散落到程序安装目录。
- 🔒 **迁移期间阻止竞态**：API 服务或推理队列运行时不允许迁移数据目录，避免上传、任务和配置在迁移过程中指向不同位置。
- 📥 **成品保持流式下载**：生成音频和编辑器渲染通过鉴权文件响应返回，调用方可以直接流式保存，不需要读取桌面端内部路径。
- 🧽 **临时上传可清理**：上传接口返回稳定 `upload_id`，任务提交后可由调用方显式删除不再使用的源文件或参考音频。

### 🎵 酷我音乐曲库

- 🎶 **第三个完整曲库**：在线资源获取新增“酷我音乐”，与网易云、QQ音乐并列，支持搜索、试听、下载、获取歌词和下载后直接进入 AI 翻唱。
- 🧾 **搜索信息完整保留**：结果映射歌曲 RID、副标题、歌手与专辑；单曲详情读取 `data.vipmusic.url`，同时展示真实格式和音质描述。
- 💿 **音质自动回退**：优先请求无损音质，失败后依次尝试 `exhigh`、`SQ` 和标准音质，不因单个档位不可用直接判定整首歌曲失效。
- 🎧 **本地代理试听**：酷我 CDN 需要 Referer 等请求头且不允许浏览器直接跨域播放；后端使用 FFmpeg 截取并转码前 `45` 秒，再以 data URI 交给播放器。
- 📦 **大文件分段下载**：携带酷我所需的浏览器请求头并使用 HTTP Range 分段获取；CDN 不支持 Range 时自动回退普通流式下载。
- 📝 **内联歌词直接解析**：优先读取 `data.lyric.lrc`，缺失时转换 `data.lyric.lrclist`；响应没有 RID 时自动回查搜索列表，不误走网易云/QQ音乐聚合歌词接口。
- 🚫 **不可播放资源提前拦截**：下载前继续通过 Content-Type、文件魔数和 FFprobe 校验真实音频，VIP、无版权、失效链接或代理错误页不会进入后续推理。

### 🛠️ 修复与可靠性

- 修复 HTTP 响应可能暴露模型、工程片段、插件和源文件绝对路径的问题；外部接口统一使用受控 DTO 与音频 URL。
- 修复批量任务前部已创建、后部校验失败后留下半批任务的问题；现改为整批验证通过后再统一入队。

### ⚠️ 安全、兼容与体积

- API 默认关闭且默认仅监听 `127.0.0.1`；切换局域网监听前应确认系统防火墙和网络可信，并妥善保存 API Key。
- CORS 允许浏览器客户端发起请求，但不会绕过 `X-API-Key`；不要把密钥写进公开网页、仓库或日志。
- 上传没有固定大小限制并不代表磁盘无限；长视频或无损多轨文件仍需预留数据盘空间，并由调用方设置合理超时。
- v0.0.22 的 NVIDIA CUDA、AMD DirectML 与 CPU 推理策略保持不变。
- 外部 API 与桌面端共用串行推理队列，不会并行占满同一张显卡；批量提交只减少调用次数，不代表任务同时推理。
