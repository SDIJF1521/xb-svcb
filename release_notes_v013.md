## v0.0.13 · 数据目录迁移 + 手动人声合并重构 + 编辑器试听导出修复

### 新增

- 💾 **用户数据目录可选择**：安装器新增「选择用户数据存储位置」页面，模型库、作品、下载素材、音频编辑工程、缓存与配置默认保存到 `.xb_xvcb`，可直接放到 D/E 盘等空间充足的位置，避免占满 C 盘。
- 🔁 **软件内数据迁移**：首页新增「数据存储位置」卡片，可查看当前目录、已用空间、所在磁盘剩余空间，并一键迁移到其它目录。
- 🧪 **迁移安全检查**：迁移前会检查目标目录可写、目标磁盘空间是否足够、目标目录是否为空或已有 XB-SVCB 数据标记，并禁止在推理任务运行/排队时迁移。
- 🧬 **真正的手动人声合并工程**：多模型「手动人声合并」不再先自动拼出完整人声，而是生成逐段可编辑素材后进入音频编辑器。
- 🎚️ **一个 AI 一条编辑轨**：手动人声合并进入编辑器后，每个参与 AI 独立成轨，轨内只包含该 AI 负责的分段音频，不再引用该模型整首推理音频。

### 优化

- ✂️ **拼接处交叉淡化贯穿试听与导出**：手动人声合并生成的分段素材会在相邻片段处预留短重叠，并写入淡入/淡出；片段试听、混音预览与导出都会走同一套时间轴渲染。
- 🎧 **声道编辑可用于试听判断**：片段试听会先保存当前工程再渲染，停止播放会清空旧音频源；后端预览改为精确声道输出，便于判断 L/R 声道摆位。
- 📤 **手动人声合并导出语义修正**：该工作流进入编辑器后，导出按钮显示为人声导出，BGM 轨默认静音，导出格式后缀会严格跟随选择的 WAV / MP3 / FLAC。
- 🧭 **旧数据目录兼容**：新默认目录为 `.xb_xvcb`；旧版本 `.xb-svcb` / `.xb_svcb` 仍会被识别，升级不会丢失已有数据。

### 修复

- 🛠️ **修复手动人声合并仍自动合并的问题**：现在手动流程会跳过自动 concat / mix，等待用户在编辑器里逐段调整和导出。
- 🎚️ **修复左右声道听感与双声道接近的问题**：声道路由改为先把片段整体合成 mono，再放到指定 L/R 声道；预览缓存随声道渲染版本更新，避免复用旧结果。
- 🧷 **修复锁定参考轨仍可编辑的问题**：锁定轨/锁定片段不再允许修改声道、时间、音量、淡化、静音和删除，避免原始音频参考轨被误改。
- 📦 **修复数据目录固定在系统盘的使用风险**：新安装和软件内迁移都可把 `.xb_xvcb` 放到其它磁盘，后续生成文件会写入迁移后的目录。

### 说明

- 数据目录迁移完成后需要重启软件；重启后所有后续模型、作品、缓存、下载素材和编辑工程都会写入新目录。
- 迁移时旧目录不会立即强删，重启到新目录并确认迁移标记后才会自动清理，降低迁移中断导致的数据风险。
- 若要验证片段声道，请优先使用片段上的「预览片段」；混音预览中未静音的 BGM 或其它轨道仍会正常在双耳播放。

### 安装

- 下载并运行 `XB-SVCB-Setup.exe` 一键安装。
- 安装时可选择 `.xb_xvcb` 用户数据目录；C 盘空间不足时建议放到 D/E 盘。
- 前端包、应用本体与安装器版本应同步更新为 **v0.0.13**。
- 40 系及以下 GPU 加速需 CUDA 12.1 运行库：https://developer.nvidia.com/cuda-12-1-0-download-archive
- 50 系（Blackwell）需 CUDA 12.8 级新版驱动；安装器会自动切换到对应栈。
