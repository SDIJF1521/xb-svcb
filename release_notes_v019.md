## v0.0.19 · 编辑器效果热更新 + 时间轴定位 + 妖狐歌词适配

> 本次更新聚焦音频编辑器的试听连续性与时间轴操作精度，并按妖狐 API 当前公开文档重新对齐在线歌词响应结构。

> [!IMPORTANT]
> v0.0.19 继续使用分卷安装包。请同时下载 `XB-SVCB-Setup.exe` 和全部 `XB-SVCB-Setup-*.bin`，并放在同一目录后再运行安装程序。

### ⚡ 音频编辑器

- 🎚️ **播放中自动更新效果**：添加、启用、关闭、删除内置效果器，或调整效果参数、音量包络和其他混音内容后，无需手动暂停再播放。
- 🔄 **当前位置热切换**：新预览在后台渲染时继续播放旧预览；渲染完成后按当时的播放时间切换音频并继续播放，不回到片头。
- ⏱️ **连续修改防抖合并**：短时间内多次修改只保留最后一次有效预览请求，过期渲染不会覆盖较新的效果状态。
- 🎯 **只响应声音变化**：使用混音内容签名区分音频与非音频修改；修改工程标题、锁定状态等不会触发无意义的音频重载。
- ⏸️ **更新期间仍可暂停**：后台刷新效果时，播放按钮继续响应暂停操作，不会被预览加载状态阻塞。

### 🎼 时间轴

- 📍 **修复选择线固定偏离鼠标**：播放头坐标改为读取标尺真实零点，不再使用与轨道标题宽度不一致的硬编码偏移。
- ↔️ **兼容横向滚动**：未滚动和横向滚动后的点击、拖动定位都使用当前 DOM 标尺位置换算，播放头与鼠标保持对齐。

### 📝 在线歌词

- 🌐 **对齐妖狐网易响应结构**：按官方返回读取 `data.lrctxt`、`data.lrc`、`data.music.lrcurl` 与 `data.music.lrc`。
- 🔗 **多候选与多层 URL 解析**：优先使用可直接解析的时间轴歌词；候选为空、下载失败或内容不是 LRC 时继续尝试下一项，歌词 URL 返回 JSON 或下一层 URL 时继续解析。
- 🎵 **QQ 免费曲库歌词回退**：QQ 免费歌曲详情不直接返回歌词时，从官方 `data.html` 提取歌曲 `mid`，再调用 `/api/music/lrc` 聚合歌词接口。
- 🧩 **保留历史响应兼容**：继续兼容顶层 `data.lrcurl`、`viplrc` 和 `lyric`，但不再把这些字段误写成网易免费接口的正式主结构。
- 👤 **歌手字段兼容**：免费接口的 `songname` 与 QQ Plus 的 `singer` 均可映射为统一歌手名称。

### 🛠️ 修复

- 修复左侧轨道标题实际宽度为 `160px`、旧坐标换算却减去 `128px`，导致播放头固定偏离鼠标 `32px` 的问题。
- 修复某个歌词字段非空但没有时间标签时提前停止，导致后续有效 `lrctxt` 或 `lrcurl` 不再尝试的问题。
- 修复只检查顶层 `lrcurl`、遗漏网易官方 `data.music.lrcurl` 的问题。
- 修复 QQ 免费曲库详情没有歌词字段时直接返回无歌词、未继续使用官方聚合歌词接口的问题。

### ⚠️ 兼容说明

- “实时效果预览”沿用现有 FFmpeg / JUCE VST3 离线渲染架构：操作后会自动渲染并热切换，不需要手动暂停/播放；实际生效延迟取决于工程长度、磁盘速度和效果器处理耗时，不是采样级零延迟 DSP。
- 新预览渲染期间旧预览会继续播放；如果渲染失败，当前预览不会被空结果替换，并会显示更新失败提示。
- QQ 聚合歌词请求继续使用用户已配置的妖狐 API Key，并受客户端统一 10 QPS 限流控制；没有可提取的歌曲 `mid` 时仍会返回无歌词提示。
- 妖狐上游字段可能继续调整；代码以当前公开的网易免费、QQ 免费、QQ Plus 与聚合歌词文档为准，同时保留有限的历史字段兼容。
