## v0.0.15 · 数据目录迁移增强 + RVC 底模预置 + 镜像源优化

### 新增

- 💾 **数据目录直接切换**：首页新增「选择目录」动作，只切换后续新文件写入位置，不复制已有数据；需要带走现有数据时仍使用「迁移数据」。
- 📊 **后台迁移进度**：数据迁移改为后台任务，前端可看到当前阶段、进度条、已复制大小和总大小，避免大目录迁移时界面像是卡住。
- 🧭 **`.sb-svcb` 数据目录规范**：新安装默认使用 `.sb-svcb`，并兼容 `.xb_xvcb`、`.sv-xvcb`、`.xb-svcb`、`.xb_svcb` 等旧目录。
- 🧩 **RVC 底模预置**：安装 RVC 环境后会把自带 ContentVec/RMVPE 预置到 rvc-python 的 `base_model`，减少首次 RVC 推理时的联网依赖。

### 优化

- 🔁 **迁移后当前会话立即生效**：后端会重定向模型库、作品库、设置与编辑工程仓储，迁移或切换后无需等到下次打开才能看到新目录状态。
- 🌐 **国内镜像默认配置**：安装器、前置依赖脚本和环境修复脚本会默认设置 `HF_ENDPOINT=https://hf-mirror.com` 与清华 PyPI 镜像；官方 PyPI 作为兜底，torch wheel 仍走 PyTorch 专用源。
- ⚙️ **RVC CUDA 栈统一**：40 系及以下 NVIDIA 环境统一使用 cu121；50 系 Blackwell 继续使用 cu128 + torch 2.7 专用栈；CPU 或不兼容显卡继续回退 CPU torch。
- 🧷 **安装器数据目录选择更安全**：选择磁盘根目录或普通非空文件夹时，会自动在其中落到 `.sb-svcb` 子目录，避免把数据散写到根目录或用户普通文件夹。

### 修复

- 🛠️ **修复 RVC 首次推理依赖外网底模的问题**：本地底模不可用时才走镜像下载，并对 RMVPE checkpoint 做格式校验/修复。
- 🧪 **修复迁移中临时缓存干扰的问题**：迁移统计和复制会跳过 `temp`、`webview` 与临时文件，减少无意义复制和被占用文件导致的失败。
- 📦 **修复数据目录指针不一致的问题**：安装器与应用内迁移会同时维护安装目录和用户 AppData 下的 `data_home.json`。

### 安装

- 下载并运行 `XB-SVCB-Setup.exe` 一键安装。
- 前端包、应用本体与安装器版本已同步更新为 **v0.0.15**。
- 40 系及以下兼容 NVIDIA GPU 默认使用 cu121 栈；50 系（Blackwell）使用 cu128 栈；CPU 或不兼容显卡会跳过 CUDA 并使用 CPU torch。
- 安装器会优先复用自带模型并自动配置镜像源；如网络仍失败，可从开始菜单运行「搭建/修复运行环境」重试。
