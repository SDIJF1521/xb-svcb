## v0.0.11 · Audio Editor Lite + 高级工作流 + 真实波形时间轴

### 新增
- 🎛️ **Audio Editor Lite 音频编辑工作台**：新增独立的轻量多轨编辑器，可从「我的作品」或本地音频创建编辑工程，支持多轨时间轴、片段拖动、边界拉伸、静音/锁定、试听、混音预览与 WAV / MP3 / FLAC 导出。
- ✂️ **播放头剪切 + 切口交叉淡化**：在时间轴或片段面板触发剪切，会把当前片段拆成左右两段，并按设置自动生成短重叠与交叉淡化，降低硬切爆音、断裂感和剪辑噪声。
- 🎚️ **片段声道分配**：每个片段可单独选择双声道、左声道或右声道；后端渲染时按声道 routing 处理，适合手动做人声摆位、左右声道对照与后期合成。
- 🧩 **编辑工程选择页**：新增音频编辑工程列表页，支持打开已有工程、导入音频新建工程、删除工程；编辑器内「退出」返回工程选择页，「放弃工程」会删除当前编辑工程后返回列表。
- 🧠 **歌声转换高级工作流**：工作台新增高级流程选择：
  - 自动混音合成（默认）
  - 自动人声合并
  - 手动人声合并（进入编辑器）
  - 自动 + 编辑器二次调整
  - 全手动编辑
- 🧬 **多模型人声合并限定**：只有多模型混合模式才显示并保留「自动人声合并 / 手动人声合并」。单模型模式会隐藏这两个选项，后端与 dev mock 也会兜底改回默认混音，避免流程语义混乱。

### 优化
- 🌊 **真实音频波形**：编辑器波形改为基于真实音频采样生成，后端按片段 `offset` 与 `end - start` 截取对应音频区间计算峰值；波形缓存纳入文件状态、偏移、片段时长与采样点数量，剪裁/拉伸后不会复用旧波形。
- 📏 **波形长度对齐时间轴**：前端按片段像素宽度动态请求对应数量的峰值，波形会铺满片段可视区域；缩放、拖动边界、修改 offset、剪切、撤销/重做后都会刷新对应波形。
- ⏩ **混音播放支持时间轴拖动**：混音预览播放时可在时间轴上拖动播放头快进/定位，播放位置会同步到音频元素，方便检查剪切点和淡化效果。
- 🕹️ **播放防抖与预览请求失效保护**：播放/暂停按钮增加防抖；新的预览请求会使旧请求失效，避免连续点击导致旧音频回填、状态错乱或播放按钮卡住。
- 🧭 **导航栏重新整理**：顶部导航减少拥挤，高频入口保留在主导航，资源获取 / 我的模型 / 我的作品等入口收纳得更清爽；「音频编辑」入口默认进入工程选择页。
- 🧪 **dev mock 完整度提升**：浏览器 `npm run dev` 环境补齐编辑工程、删除工程、工作流规范化、波形长度模拟、局部重推理等 mock 行为，前端调试更接近桌面环境。

### 修复
- 🛠️ **自动 + 编辑器二次调整无效果**：修复该高级流程完成后停留在结果页的问题；现在生成完成会自动创建编辑工程并跳转到音频编辑工作台，失败时给出明确提示。
- 🧷 **局部重推理片段过短保护**：片段短于 1 秒时前端禁用替换按钮，后端同步拦截，避免剪得过碎导致 SVC/RVC 推理不稳定。
- 🗑️ **编辑工程删除按钮 dev 下无效**：删除按钮增加事件阻止、层级和删除中状态，避免点击被工程卡片主区域抢走；删除失败会提示并刷新列表。
- 🚪 **编辑器退出/放弃流程**：修复退出目标不明确的问题；退出统一回到工程选择页，放弃工程会删除当前工程并返回列表。
- 🎼 **人声合并流程误用于单模型**：单模型提交人声合并 workflow 时，前端、mock 与后端都会归一化为自动混音，避免单模型任务跳过伴奏混音或输出语义错误。

### 说明
- 「自动人声合并 / 手动人声合并」只适用于多模型混合：因为它们本质上是在多段、多模型人声之间做合并与交叉淡化，单模型不需要额外的人声合并阶段。
- 「全手动编辑」不会启动自动推理，会直接用所选音频创建编辑工程；「自动 + 编辑器二次调整」会先跑完整自动生成，再进入编辑器做二次调整。
- 真实波形依赖桌面后端读取音频文件；纯浏览器 dev 环境无法读取本机音频二进制，因此使用按时间轴宽度对齐的模拟波形。

### 安装
- 下载并运行 `XB-SVCB-Setup.exe` 一键安装（约 2 GB，含分离 / so-vits-svc / RVC / 模型站 / 音频编辑器组件）。
- 前端包、应用本体与安装器版本已同步更新为 **v0.0.11**。
- 40 系及以下 GPU 加速需 CUDA 12.1 运行库：https://developer.nvidia.com/cuda-12-1-0-download-archive
- 50 系（Blackwell）需 CUDA 12.8 级新版驱动；安装器会自动改装 cu128 + torch 2.7 栈。
