## v0.0.17 · 音频编辑器效果器 + JUCE VST3 Host + 音频剪贴板

### 新增

- 🎛️ **音频编辑器效果器链**：片段支持叠加混响、降噪、噪声门、压缩、EQ、高通、低通、延迟、合唱、限幅、增益等内置效果器，适合在人声修补、伴奏处理和音效轨道上做基础后期。
- 📈 **音量包络**：片段可启用多点音量包络，渲染时按时间线插值，便于做局部压低、渐强、句尾修整和人声贴合。
- 🧩 **JUCE VST3 插件 Host**：外部效果器改为 `Python（业务逻辑、AI、界面） -> C++（JUCE VST3 Host） -> VST3 Plugin GUI` 架构，支持 VST3 插件检查、离线渲染、原生插件 GUI 打开和插件 state 回写。
- 🪟 **插件窗口弹窗**：音频编辑器新增插件窗口入口，可在前端选择插件路径、检查插件状态，并打开插件原生界面调整参数。
- 📋 **复制片段/音轨音频**：选中的片段或整条音轨可以渲染为音频文件并复制到系统剪贴板，便于粘贴到资源管理器或其他音频软件中继续使用。
- 📥 **粘贴音频到音轨**：支持把软件内复制的音频，或资源管理器复制的 WAV / MP3 / FLAC / M4A 等音频粘贴回编辑器音轨；多个音频文件会从播放头开始顺序铺开。
- 🖱️ **单实例桌面窗口**：程序未关闭时再次点击 `XB-SVCB.exe`，会唤起已有窗口并置于 Windows 前台，不再重复打开多个界面。

### 优化

- 🧱 **前端组件化**：编辑器导入音频弹窗与插件窗口弹窗拆分为 `EditorImportTrackDialog.vue` 和 `EditorPluginDialog.vue`，主编辑器页面只保留时间轴编排、选区和状态管理。
- 🧭 **插件系统职责分离**：Python 只负责编排业务、AI 推理和编辑工程状态；C++ JUCE Host 负责加载 VST3、渲染插件、显示原生 GUI 和同步插件 state，降低插件兼容问题对主程序的影响。
- 📦 **安装器 Host 校验**：发布构建会强制检查 `engines/juce-vst3-host/xb-juce-vst3-host.exe` 是否存在，缺失时停止生成安装器，避免用户安装后插件系统不可用。
- 🎵 **网易云音频下载兜底**：音乐 API 返回 `url` 和 `musicurl` 时会逐个校验，哪个可播放就用哪个；只有两个都不可用时才判定下载失败。若接口只返回其中一个字段，则只校验这个字段。
- 🧾 **架构文档更新**：README 架构图补充了 pywebview 单实例桌面壳、Python 本地 API、音频剪贴板、内置效果链、JUCE VST3 Host 和 VST3 Plugin GUI 的关系。

### 修复

- 🛠️ **修复插件效果污染 AI 人声的问题**：局部重推理会使用原始片段裁剪作为模型输入，清理旧 dry/output 缓存，替换片段后移除插件类效果并记录到 metadata，避免 VST3 插件处理结果被误当作新的模型干声。
- 🧹 **清理推理旧输出**：SVC / RVC 推理前会清理同路径旧输出，减少缓存残留导致结果看起来像被其他音频覆盖的问题。
- 🔁 **重推理缓存键增强**：局部重推理输出会纳入源文件、片段位置、模型和推理参数，减少不同片段或不同参数复用旧结果的风险。
- 🌐 **修复单个音乐 URL 字段被误判不可播放的问题**：网易云接口只返回 `url` 或只返回 `musicurl` 时，不再因为另一个字段缺失而直接失败。

### 兼容说明

- VST3 插件需要与 Host 架构匹配，例如 64 位 Host 搭配 64 位插件。
- v0.0.17 首先面向 VST3 **效果插件**，乐器插件和 MIDI 专用插件不作为音频编辑器目标。
- 最终用户安装包会携带编译好的 `xb-juce-vst3-host.exe`，不需要在用户机下载 JUCE SDK，也不需要现场编译 C++ Host。
- 源码开发者如需重新构建 Host，可设置 `XB_JUCE_DIR` 指向本地 JUCE 源码目录，再运行 `native/juce-vst3-host/build.ps1`。

### 安装

- 下载并运行 `XB-SVCB-Setup.exe` 一键安装。
- 前端包、应用本体与安装器版本已同步更新为 **v0.0.17**。
- 安装器会随包释放 JUCE VST3 Host；如安装后插件系统不可用，请检查安装日志中关于 `xb-juce-vst3-host.exe` 的记录。
- 40 系及以下兼容 NVIDIA GPU 默认使用 cu121 栈；50 系（Blackwell）使用 cu128 栈；CPU 或不兼容显卡会跳过 CUDA 并使用 CPU torch。
