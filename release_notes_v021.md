## v0.0.21 · 插件并行交互 + 音频合并 + 块级实时播放

> 本次更新集中改善音频编辑器的连续工作体验：VST3 插件窗口不再阻断主编辑器操作，片段可以直接渲染合并，插件参数变化由 JUCE 声卡回调在下一个音频块生效。

> [!IMPORTANT]
> v0.0.21 继续使用分卷安装包。请同时下载 `XB-SVCB-Setup.exe` 和全部 `XB-SVCB-Setup-*.bin`，放在同一目录后再运行安装程序。

### 🎛️ 插件窗口

- 🪟 **主界面与插件并行操作**：插件控制窗口改为非模态、可拖动并允许点击底层编辑器；打开 VST3 原生 GUI 后仍可操作播放、播放头和时间轴。
- 📌 **原生窗口保持可见**：JUCE Host 插件窗口启用置顶和窗口阴影；主程序重新获得焦点时，插件 GUI 不再消失或被误认为已关闭。
- 🔄 **轻量状态同步**：播放期间 Host 只写传输、设备、峰值与轻量插件信息，不再反复抓取大型 state 或遍历数百个参数；暂停后刷新参数，关闭窗口时再安全抓取完整 state。
- 📊 **插件频谱与 VU 实时运动**：打开插件时生成目标插件之前的监听信号，播放期间把音频块送入 GUI 所属的同一个插件实例，Pro-Q 等插件的频谱、VU 和增益衰减表可随播放运动。
- 🎧 **处理结果直接可听**：后端同时准备“插件前目标片段”和“移除目标片段后的工程底轨”；JUCE 在声卡线程中处理目标插件并与底轨混合，GUI 绑定的就是实际发声实例。
- 🧱 **块级低延迟处理**：Host 首选 `128 samples` 缓冲；WASAPI 共享模式、ASIO 或具体声卡不支持时使用设备实际缓冲，并在插件面板显示设备名、块大小和输出延迟。
- 📐 **真实回调块大小**：插件每次只接收声卡当前回调的有效样本视图；例如设备回调为 `480 samples` 时，Host 不再把内部 `8192 samples` 缓冲容量误报给 `processBlock`。
- 🔁 **跨采样率安全播放**：工程素材与声卡采样率不一致时在回调中进行线性采样率转换，避免 `44.1 kHz` 工程在 `48 kHz` 设备上发生速度、音高或重复采样异常。
- 🎯 **播放头精确同步**：前端每 `120ms` 同步播放、暂停和定位；混音使用全局时间，单片段试听只驱动对应插件片段，播放头位于片段外时送入静音。
- 🧹 **会话自动清理**：用户从插件自身关闭原生窗口后，编辑器会识别进程结束并停止轮询；退出编辑器时也会关闭残留会话。
- 🚫 **防止重复窗口**：同一会话已打开时禁用再次打开，避免多个原生窗口竞争同一插件状态文件。

### 🔗 音频片段合并

- 🎼 **合并下一片段**：片段检查器和时间轴工具栏新增合并入口，可将所选片段与同轨紧邻的后一片段合成一个新片段。
- 🎚️ **真实渲染而非拉长边界**：合并会处理两段之间的空白、重叠、片段音量、淡入淡出、音量包络、声道和效果器，再按工程采样率输出新的 WAV。
- 🧾 **来源可追踪**：合并片段记录来源片段 ID、名称、文件和原始时间范围；相同角色会继续保留，不同角色不会错误继承第一段角色。
- ↩️ **支持撤销**：来源片段替换操作进入工程历史，可通过编辑器撤销恢复；新音频保留在工程媒体目录供重做和后续编辑使用。
- 🔒 **合并前完整校验**：锁定轨道、锁定片段、缺失源文件、跨轨选择和非连续选择会给出明确错误，不生成不完整音频。

### ⚡ 播放中实时调整

- 🎛️ **插件参数下一块生效**：旋转原生插件 GUI 参数时不再等待整段 FFmpeg 预览；同一插件实例在声卡回调中持续执行 `processBlock`，参数变化在下一音频块进入可听输出。
- 🧭 **原生播放头回传**：播放、暂停和定位通过独立控制文件同步，JUCE 回传实际采样位置、结束状态、峰值、设备延迟和 xrun 计数，前端时间轴跟随真实输出位置。
- 📈 **完整插件播放上下文**：每个处理块向插件提供 `AudioPlayHead`，包含播放状态、样本/秒位置、`120 BPM`、4/4 拍、PPQ 和小节起点，修复依赖宿主时间信息的频谱、滚动曲线与节拍同步效果不更新。
- 🪟 **界面先于音频启动**：插件窗口先进入 Windows 消息循环并完成首轮初始化，再启动声卡和 DSP 回调，避免 Pro-Q 等插件的 GUI 初始化锁与音频线程互相等待而白屏或未响应。
- 🛡️ **可靠回退**：只有收到原生声卡握手后才停用 HTML Audio；没有可用输出设备或握手失败时继续使用预渲染播放，独立输出开关防止两套播放器同时发声。
- 🔊 **单一可听输出**：进入原生实时播放时强制暂停两只 HTML Audio；多个插件会话同时存在时只有最近激活的会话可向声卡输出，其余会话继续处理仪表但保持静音。
- 🔇 **静音插件保护**：乐器、MIDI、零输入或零输出插件可以打开 GUI，但不会接管人声效果输出；有效效果器连续有输入却输出静音超过插件延迟加约 `500ms`，或返回 NaN/Inf 时自动旁通为干声，恢复有效输出后自动解除。
- 🎚️ **拖动即更新**：片段音量、淡入淡出、音量包络和内置效果参数在拖动时以 `120ms` 防抖保存，不必等到松开后才开始更新预览。
- 📥 **最新状态排队**：正在渲染时继续播放旧预览；后续参数变化只保留最新状态排队，不再取消已开始的渲染，避免持续拖动时长期听不到变化。
- 🔀 **双播放器热切换**：新预览完成后从当前播放位置启动备用播放器，并以约 `90ms` 交叉淡化替换旧预览，降低切换时的跳音和瞬时断声。
- ⏱️ **更低响应延迟**：混音刷新防抖由 `180ms` 调整为 `90ms`，插件传输位置同步由 `850ms` 调整为 `120ms`。
- ⏸️ **播放控制始终可用**：后台渲染和插件状态同步期间仍可暂停、定位或退出；过期结果不会在停止播放后重新抢占播放器。
- 🛠️ 修复
- 修复点击 XB-SVCB 主窗口后插件 UI 被遮挡、看起来像自动关闭的问题。
- 修复 Element Plus 模态层拦截播放按钮和时间轴点击，导致插件参数编辑与音频播放无法并行的问题。
- 修复持续旋转插件旋钮时，每次 state 变化都会取消前一次渲染，预览可能一直停留在旧参数的问题。
- 修复单播放器直接替换 `src` 时出现明显断声、跳音和播放位置体感不连续的问题。
- 修复插件原生窗口自行关闭后，前端仍保留已打开状态并继续无效轮询的问题。
- 修复插件 GUI 实例从未执行 `processBlock`，导致频谱、VU 和动态增益表始终没有声音波动的问题。
- 修复把 VST3i 乐器插件当作人声效果器处理时，因没有 MIDI 音符和音频输入而让整段人声变成静音的问题。
- 修复部分插件音频总线不可用、持续返回零或非法采样后，原生输出仍停用 HTML Audio 导致完全无声的问题。
- 修复旧工程音轨 ID 异常时目标片段没有从底轨移除，干声与插件处理声叠加形成两个声音的问题。
- 修复双播放器交叉淡化或状态更新滞后时，切换 JUCE 输出没有彻底暂停 HTML Audio 导致重复发声的问题。
- 修复 Host 未提供 `AudioPlayHead`，导致部分插件虽然收到音频但频谱、时间曲线或节拍同步视图不运动的问题。
- 修复把内部缓冲容量而非声卡本次有效样本数交给 `processBlock`，导致部分插件实时输出为零、听不到效果、分析曲线间歇消失或调参后停止更新的问题。
- 修复声卡线程在插件窗口进入消息循环前启动，导致 FabFilter Pro-Q 4 偶发白屏、未响应或只在点击界面后才开始绘制的问题。
- 修复播放期间高频抓取大型插件 state 和全部参数，占用插件 GUI 消息线程并可能让频谱停止刷新的问题。

### ⚠️ 兼容与性能

- VST3 原生 GUI 打开且声卡握手成功时，当前目标插件使用 JUCE 块级实时输出；无插件会话、内置 FFmpeg 效果调整和离线导出仍使用预渲染链路。
- VST3i 乐器与 MIDI 插件不是当前人声效果机架的目标；Host 会保留其 GUI，但实时播放和离线导出跳过其音频处理并保留原音。
- 实时监听会烘焙目标插件之前的效果、片段音量、包络与淡化；目标插件之后的效果在最终离线导出时完整处理，不会在实时目标支路重复计算。
- 缓冲大小由声卡驱动决定。Host 请求 `128 samples`，WASAPI 共享模式常见实际值为 `480 samples / 10 ms`；界面显示实际值，不把请求值当作设备值。
- 新预览渲染期间旧音频会继续播放，渲染完成后才交叉淡化切换；渲染失败不会用空音频替换当前播放。
- 合并会把所选片段的片段级效果烘焙进新 WAV；目标音轨的音量不会烘焙，避免合并后被重复放大或衰减。
- 当前 UI 提供“所选片段 + 后一片段”的连续合并；后端接口支持多个同轨连续片段，为后续多选合并保留扩展空间。
- 本版本仅支持 **64 位 Windows VST3 音频效果器**，`.vst3` 可以从用户选择的任意目录加载；暂不支持 VST2 `.dll`、32 位插件、CLAP、AAX、AU 和需要 MIDI 音符驱动的 VST3i 乐器。插件是否提供原生编辑器、外部侧链或特殊多总线能力取决于插件自身实现。
