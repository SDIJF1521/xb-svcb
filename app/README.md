# XB-SVCB 应用本体

版本：`0.0.22`

这里包含打包进 `XB-SVCB.exe` 的桌面应用壳、本地 API 桥接、领域服务和基础设施适配层。

## 运行结构

- `app/main.py` 启动 pywebview 桌面壳。
- `app/config.py` 提供应用元信息、路径与运行环境位置。
- 重型 AI 依赖不打进应用本体，由安装器放到 `.venv-svc`、`.venv-rvc`、`.venv-seedvc`、`.venv-ddsp`、`.venv-uvr`、`.venv-hub` 等隔离环境中。

## v0.0.22 重点

- 应用本体版本同步到 `0.0.22`，本地 API、前端、两份锁文件和安装器使用同一版本号。
- 推理设备统一支持 `auto`、`cuda`、`rocm`、`directml` 与 `cpu`，并按每个隔离环境报告真实可用后端。
- So-VITS-SVC、RVC、SeedVC、DDSP-SVC 与 F0 worker 增加 Windows AMD DirectML 路径；DirectML 不支持的复数 STFT 运算保留在 CPU，实数张量返回 GPU。
- UVR worker 使用 `audio-separator` 的 DirectML 后端；VR `.pth` 模型走 `torch-directml`，MDX `.onnx` 模型走 `onnxruntime-directml`。
- 显式选择 CUDA、ROCm 或 DirectML 时严格校验实际设备，失败会返回明确错误，不再静默回退 CPU。

## v0.0.21 重点

- 应用本体版本同步到 `0.0.21`，本地 API、前端、两份锁文件和安装器使用同一版本号。
- 外部插件当前仅支持 64 位 Windows VST3 音频效果器；插件路径不受系统默认目录限制。VST2 `.dll`、32 位插件、CLAP、AAX、AU 和需要 MIDI 音符的 VST3i 乐器暂不进入人声音频效果链。
- 音频编辑服务新增同轨连续片段合并：先按时间线渲染片段的空白、重叠、音量、淡化、包络和效果，再以一个新 WAV 片段替换来源片段。
- 合并结果保留来源片段 ID、文件和时间范围，写入工程撤销历史，并清空旧波形缓存。
- JUCE Host 播放期间只轮询轻量运行状态，暂停后刷新参数，关闭窗口时再抓取完整 state；插件原生窗口关闭后自动清理会话，未变化状态不重复写工程。
- 插件窗口保持置顶并与主编辑器并行交互，点击播放、时间轴或其他主窗口控件不会结束插件 GUI 会话。
- 打开插件时会生成目标插件前的片段信号和移除目标片段后的工程底轨；JUCE 声卡回调处理 GUI 所属插件实例并混合底轨，参数在下一音频块进入可听输出。
- 原生会话回传设备名、实际采样率、缓冲大小、延迟、播放位置和 xrun；只有声卡握手成功后前端才切换原生输出，否则继续使用 HTML Audio 回退。
- 插件兼容检查会排除乐器/MIDI 与无音频总线插件；实时效果器持续输出静音或非法采样时自动干声旁通，避免打开插件后整段无声。
- 实时底轨按目标片段 ID 从所有音轨剔除目标，兼容旧工程的异常音轨引用；多个插件会话使用全局独占可听输出，避免干声/湿声或多 Host 重复发声。
- 声卡回调只把本次有效样本数交给 VST3 `processBlock`，并在插件 GUI 进入消息循环后启动音频，避免部分插件无效果、无频谱或调参后停止刷新的问题。

## v0.0.20 重点

- 应用本体版本同步到 `0.0.20`，本地状态 API、前端、锁文件和安装器使用同一版本号。
- 新增 DDSP-SVC 6.3 Engine/worker、Rectified Flow 推理、模型导入、F0、采样步数、说话人和共振峰偏移参数。
- DDSP worker 按模型 YAML 读取 `f0_min` / `f0_max`，并使用 ContentVec、RMVPE 与 PC-NSF-HiFiGAN 2025.02 底模链路。
- UVR worker 严格执行用户选择的 `cuda` / `cpu`；CUDA 环境不可用时明确失败，不再静默回退 CPU，运行日志记录实际分离设备。
- 音频编辑服务支持普通 TXT 自动定时切句：优先吸附静音中心，缺少静音点时按歌词长度比例分配；LRC 继续按时间戳切分。
- 编辑工程领域结构正式包含角色列表，多角色分配和时间轴模板元数据可持久化保存。
- 消息中心改为全局状态，作品任务与环境状态可跨页面和多前端窗口同步已读状态。
- 桌面壳配合新版前端在主题动画结束后同步原生标题栏与窗口边框，避免暗色切换期间窗口外观提前跳变。

## v0.0.19 重点

- 应用本体版本同步到 `0.0.19`，本地状态 API、前端与安装器使用同一版本号。
- 音乐服务按妖狐当前官方响应读取 `data.lrctxt`、`data.lrc`、`data.music.lrcurl` 与 `data.music.lrc`，并继续尝试其他可解析候选。
- 歌词 URL 支持返回纯 LRC、JSON 歌词字段或下一层 URL；无效的非空候选不会阻断后续字段。
- QQ 免费详情没有歌词字段时，从官方 `data.html` 提取歌曲 `mid`，再请求 `/api/music/lrc` 聚合歌词接口。
- 免费接口的 `songname` 与 QQ Plus 的 `singer` 统一映射为歌手名称；顶层 `lrcurl`、`viplrc` 和 `lyric` 仅作为历史兼容。
- 音乐服务测试覆盖网易嵌套歌词地址、QQ `mid` 提取，以及聚合歌词的正式地址和 `key` / `mid` / `type` 参数。

## v0.0.18 重点

- 应用本体版本同步到 `0.0.18`，本地状态 API、前端与安装器使用同一版本号。
- 新增 SeedVC 引擎、worker 与运行环境探测，支持 checkpoint + YAML 配置和推理时目标音色参考音频。
- 模型导入、模型站清单、任务参数和跨框架引擎路由均识别 `seed-vc`。
- 模型站下载支持 `.part` 断点续传、HTTP Range 与自动重试；清单声明的主模型、配置、扩散模型和 index 全部完整后才导入本地模型库。
- 主题媒体桥接支持选择图片或 MP4，将文件复制到用户数据目录并按需返回 data URI。
- 音乐服务适配妖狐 API `V2.1.3.8`：不再发送已废弃的 `g` 参数，并兼容 `vipmusicurl` 与 URL 型歌词。
- 安装器运行时路径新增 `.venv-seedvc` 与 `engines/seed-vc`，应用状态页可显示 SeedVC 环境是否就绪。

## v0.0.17 重点

- 应用本体版本同步到 `0.0.17`，本地 API 配置返回新版版本号，便于前端、安装器和发布包统一识别。
- 音频编辑服务新增片段/音轨音频复制到系统剪贴板，以及从剪贴板读取音频文件并按播放头粘贴到选中/新建音轨。
- 剪贴板音频在 Windows 优先使用 CF_HDROP 文件列表，同时兼容文本路径和 `file://` URI 回退；支持 WAV / MP3 / FLAC / M4A 等常见音频文件。
- 效果链新增音量包络、混响、降噪、噪声门、压缩、EQ、高通、低通、延迟、合唱、限幅、增益等片段级处理，并通过 JUCE VST3 Host 桥接外部插件效果器。
- 外部插件效果器改为通过原生 JUCE Host 加载 VST3，并支持插件 GUI 打开、检查、离线渲染和 state 回写。
- 局部重推理会清理旧 dry/output 缓存，替换片段后移除插件类效果并记录 metadata，避免 VST3 插件状态污染模型生成的人声。
- 桌面本体启用单实例入口：程序未关闭时再次启动 exe，会通知已有窗口置前并退出新进程。

## v0.0.16 重点

- 应用本体版本同步到 `0.0.16`，本地 API 配置返回新版版本号，便于前端、安装器和发布包统一识别。
- 打包进 `XB-SVCB.exe` 的前端包含新版主题系统、自定义主题编辑器、多角色管理和时间轴模板。
- 编辑工程 JSON 可保存角色列表以及轨道/片段上的角色标记；旧工程未包含这些字段时仍按原结构读取。
- 安装器环境搭建脚本修复 UVR 部署时 GPU 用户被替换为 CPU 版 PyTorch 的问题，应用本体继续复用隔离子环境。

## v0.0.15 重点

- 用户数据目录默认使用 `.xb_svcb`，并兼容 `.sb-svcb`、`.xb_xvcb`、`.sv-xvcb` 等旧目录。
- 数据目录切换/迁移后会在当前会话内重定向模型、作品、设置和编辑工程仓储。
- 数据迁移改为后台任务，后端提供进度、阶段、已复制大小和结果状态。
- RVC worker 会优先使用自带 hubert/rmvpe 底模，并在必要时走 HuggingFace 镜像兜底。
