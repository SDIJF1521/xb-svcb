# XB-SVCB 应用本体

版本：`0.0.17`

这里包含打包进 `XB-SVCB.exe` 的桌面应用壳、本地 API 桥接、领域服务和基础设施适配层。

## 运行结构

- `app/main.py` 启动 pywebview 桌面壳。
- `app/config.py` 提供应用元信息、路径与运行环境位置。
- 重型 AI 依赖不打进应用本体，由安装器放到 `.venv-svc`、`.venv-rvc`、`.venv-uvr`、`.venv-hub` 等隔离环境中。

## v0.0.17 重点

- 应用本体版本同步到 `0.0.17`，本地 API 配置返回新版版本号，便于前端、安装器和发布包统一识别。
- 音频编辑服务新增片段/音轨音频复制到系统剪贴板，以及从剪贴板读取音频文件并按播放头粘贴到选中/新建音轨。
- 剪贴板音频在 Windows 优先使用 CF_HDROP 文件列表，同时兼容文本路径和 `file://` URI 回退；支持 WAV / MP3 / FLAC / M4A 等常见音频文件。
- 效果链新增音量包络、混响、降噪、噪声门、压缩、EQ、高通、低通、延迟、合唱、限幅、增益等片段级处理，并通过 JUCE VST3 Host 桥接外部插件效果器。
- 外部插件效果器改为通过原生 JUCE Host 加载 VST3，并支持插件 GUI 打开、检查、离线渲染和 state 回写。
- 局部重推理会清理旧 dry/output 缓存，替换片段后移除插件类效果并记录 metadata，避免 VST3 插件状态污染模型生成的人声。
- 桌面本体启用单实例入口：程序未关闭时再次启动 exe，会通知已有窗口置前并退出新进程。

## v0.0.16 重点

- 应用本体版本同步到 `0.0.16`，本地 API 配置返回新版版本号，便于前端、安装器和发布包统一识别。
- 打包进 `XB-SVCB.exe` 的前端包含新版主题系统、自定义主题编辑器、多角色管理和时间轴模板。
- 编辑工程 JSON 可保存角色列表以及轨道/片段上的角色标记；旧工程未包含这些字段时仍按原结构读取。
- 安装器环境搭建脚本修复 UVR 部署时 GPU 用户被替换为 CPU 版 PyTorch 的问题，应用本体继续复用隔离子环境。

## v0.0.15 重点

- 用户数据目录默认升级为 `.sb-svcb`，并兼容 `.xb_xvcb`、`.sv-xvcb` 等旧目录。
- 数据目录切换/迁移后会在当前会话内重定向模型、作品、设置和编辑工程仓储。
- 数据迁移改为后台任务，后端提供进度、阶段、已复制大小和结果状态。
- RVC worker 会优先使用自带 hubert/rmvpe 底模，并在必要时走 HuggingFace 镜像兜底。
